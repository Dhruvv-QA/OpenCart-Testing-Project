# Bug Reports – OpenCart Testing Project

This folder contains defects identified during manual testing of the OpenCart e-commerce application.

## Testing Scope
- Functional Testing
- Negative Testing
- Edge Case Testing
- Usability Testing

## Modules Covered
- Login
- Cart
- Product Page
- Product Comparison

## Total Bugs Identified: 4

| Bug-01 | Out of stock product can be added to cart | High | Functional |
| Bug-02 | Negative quantity shows success message but product not added | Medium | Functional/UI |
| Bug-03 | Product comparison page not accessible via UI | High | Usability |
| Bug-04 | Mandatory option without selectable values blocks add to cart | High | Functional |

## Bug Reporting Approach
Each bug includes:
- Detailed description
- Steps to reproduce
- Expected vs Actual result
- Severity & Priority
- Supporting screenshots

## Tools Used
- JIRA (Bug tracking)
- MS Excel (Test cases & scenarios)
- GitHub (Documentation & version control)

## Objective
To validate application functionality and identify defects impacting user experience and system behavior.

## Key Learning
- Improved defect identification skills
- Learned real-time bug reporting process
- Understood user experience impact of defects
